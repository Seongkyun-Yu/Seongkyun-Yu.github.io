S3에 프로젝트 업로드 하기

AWS S3 서비스에 접속해서 버킷 만들기를 누릅니다.


버킷 이름을 작성하고 다음으로 넘어갑니다.


기본 암호화에 체크한 후 다음으로 넘어갑니다.


모든 퍼블릭 액세스 차단의 체크를 풀고 다음으로 넘어가서 버킷 만들기를 마무리합니다.


버킷 속성에서 정적 웹사이트 호스팅을 누르고 인덱스 문서와 오류 문서에 index.html을 작성 후 저장합니다.

여기서 생성된 엔드포인트 주소로도 업로드할 프로젝트에 접근할 수 있게 됩니다.


앞서 버킷을 퍼블릭으로 설정은 했지만 아직 어떠한 기능을 공개할지 정하지 않았기 때문에 그에 필요한 정책을 만들어야 합니다.

권한 탭을 누르고 버킷 정책에서 정책 생성기를 클릭합니다.


Select Type of Policy에서 S3 Bucket Policy를 선택합니다.

Principal에는 적당히 규칙명을 적어주고 Actions에서 Get Object를 선택합니다.


Amazon Resource Name (ARN)에는 arn:aws:s3:::<bucket_name>/<key_name> 형식의 문자열을 만듭니다.
저의 경우 arn:aws:s3:::studyfe.ml/*으로 작성 했습니다.
그 후 Add Statement와 Generate Policy를 눌러 정책을 만듭니다.


생성된 정책을 버킷에 붙여넣고 저장합니다.


개요 탭을 누르고 업로드를 클릭하여 프로젝트를 업로드 합니다.


CloudFront

S3에서 제공해주는 엔드포인트로도 앱을 배포할 수 있지만 CloudFront를 사용하는 이유는 다음과 같습니다.

  • HTTPS 배포
  • 전세계 엣지 로케이션을 이용한 더 빠른 속도
  • S3 직접 공유보다 저렴한 비용

S3에서 제공하는 주소는 HTTP로 보안에 취약합니다. CloudFront는 이를 쉽게 HTTPS로 배포할 수 있게 도와줍니다.

S3를 이용해 직접 배포할 경우 S3 버킷을 설정한 지역의 저장소로 전세계 사용자가 요청을 보냅니다. 따라서 먼 지역의 유저일 수록 속도가 느려질 것입니다. 하지만 CloudFront를 사용한다면 전세계 엣지 로케이션에서 캐싱을 해두고 유저가 가까운 엣지에 요청을 보내도록 합니다. 따라서 S3보다 속도가 더 빠릅니다.

또한 CloudFront의 요금이 S3 요금보다 저렴하기 때문에 비용절감 효과가 있습니다.



CloudFront 서비스로 들어가 Create Distribution을 선택한 후 Get Started를 누릅니다.



Origin Domain Name에 방금 만든 버킷을 선택합니다.

Viewer Protocol Policy은 Redirect HTTP to HTTPS로 설정하고

Default Root Object에는 index.html을 작성한 후 Create Distribution을 누릅니다
